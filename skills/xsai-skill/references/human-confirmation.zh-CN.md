# 人工确认边界

Operator gateway 的存在，是为了让 AI 不打开后台也能做事，但有些动作仍然必须由人来做最后确认。

## 始终需要人工确认

- 确认未支付订单已支付
- 确认钱包充值已到账
- 确认 API 充值已到账
- 批准或拒绝代理申请
- 批准或拒绝 API 接入申请

这些确认动作在后台 operator alerts inbox 中完成，不能只靠外部 skill。

## 允许 AI 执行的动作

- 在 token 允许时起草和发布内容
- 更新安全的商品字段
- 切换上下架状态
- 预览库存导入
- 提交库存导入
- 读取告警
- ack 或 snooze operator 可见告警
- 生成带理由和置信度的审核建议

## 库存规则

- 永远先 preview 再 commit
- AI 可以知道数量、重复项、导入前库存和导入后库存
- API 不得把原始卡密回传给 AI

## 支付规则

- AI 只读取支付汇总
- AI 可以提示未处理告警
- 最终是否到账由人确认
